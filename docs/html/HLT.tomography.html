class HLT.tomography.constraint_matrix.ConstrainMatrix(sys_size, range_constraints, range_hamil_terms)

Bases: object

Parameters

  • sys_size (int) – number of qubits to reconstruct

  • range_constraints (int) – the maximum locality of the constraints in the constraint matrix (should be bigger than range_hamil_terms, usually taken as range_constraints=range_hamil_terms+1)

  • range_hamil_terms (int) – the maximum locality of the Hamiltonians which assumed to be in the GH. These Hamiltonian will define the constraint matrix.

  • hamil_terms – list of Paulis used in the constraint matrix

  • constraints – list of constraints (Paulis) used in the constraint matrix

  • operator_matrix – a matrix which contains the operators in the constraint matrix entries as defined in the main paper. Since the Hamiltonians and constraint are Pauli operators with k1 and k2 maximal locality, respectively, these entries, which are commutation relations between constraints and hamiltonians, are at most (k1+k2-1)-local Paulis (usually k2=k1+1).

  • constraint_matrix – the constraint matrix filled with expectation value from experiment or simulation data (after get_constraint_matrix is called).

  • sys_size – number of qubits to reconstruct

  • range_constraints – the maximum locality of the constraints, which are taken to be Pauli operators, in the constraint matrix (should be bigger than range_hamil_terms, usually taken as range_constraints=range_hamil_terms+1).

  • range_hamil_terms – the maximum locality of the Hamiltonians which assumed to be in the GH. These Hamiltonian will define the constraint matrix.

Return type
